# DayLog

用自然语言随手记录每天做了什么，一键生成日报 / 周报 / 季度总结 / 年度总结。本地优先、数据全在自己电脑上，完全断网也能用。

---

## 安装

> Windows 10 / 11，64 位。

**方式一：安装版（推荐）**

1. 下载 `DayLog_0.1.0_x64-setup.exe`。
2. 双击运行，按提示下一步即可（安装到当前用户，无需管理员权限）。
3. 从开始菜单打开 DayLog。

安装包内置了运行所需的 WebView2 组件，**全程不需要联网**，适合内网 / 离线电脑。

**方式二：绿色免安装版**

1. 下载 `DayLog.exe`，放到任意文件夹。
2. 想让数据跟着程序走（比如放 U 盘里随身带），在 `DayLog.exe` 旁边新建一个空文件夹 `data`——此后所有记录都存在这个文件夹，删掉它即清空全部数据。
3. 双击 `DayLog.exe` 启动。

> 绿色版依赖系统已自带的 WebView2（Win11 自带，多数 Win10 也有）。若打开报缺组件，改用安装版即可。

## 使用说明

### 1. 记录

打开就是「记录」页。直接用大白话写你做了什么，**没有任何格式要求**，写完按 `Ctrl+Enter` 保存。

- 想给记录打标签 / 归类，在文字里写 `#标签` 或 `@项目` 即可，例如：
  > 上午和产品对齐了 Q3 方向 #会议 @DayLog
  其中 `#会议` 会成为标签、`@DayLog` 会成为项目，用于后续筛选和报告分组。**正文照样原样保留**。
- 名称到第一个空格或标点为止。例如 `@DayLog：开始打包` 的项目名是 `DayLog`。
- 输入 `#` 或 `@` 时会弹出你用过的标签 / 项目供选择，避免写串。

**一次记多天**：把好几天的内容一起贴进来，点「AI 拆分多日…」，AI 会按日期拆好、你确认无误后再入库（需先配置 AI，见第 4 节）。无法确定日期的条目会标黄并默认归到今天，可手动改。

### 2. 回顾

「回顾」页是一个月历，每天做了什么一目了然，点某一天可展开当天全部记录、也能补记。

- 顶部可切换年 / 月（也支持 `←` `→` 翻月、`PageUp` `PageDown` 翻年）。
- 右上搜索框：直接输文字按内容搜；输入 `#` 或 `@` 可按标签 / 项目筛选，能混着写（如 `联调 @DayLog`）。搜索不区分大小写。

### 3. 生成报告

「报告」页选好类型（日报 / 周报 / 季度 / 年度）和时间范围（用日期选择器），即可生成：

- **直接整理**（默认，离线）：把这段时间的记录按模板填充，原文呈现、按天 / 标签分组。
- **AI 总结**（需配置 AI）：让模型把这段记录提炼成总结正文。

生成后可复制，或导出为 `.md` / `.html`，也能打印成 PDF。导出文件名自动带周期编号，如 `Weekly_2026-W24`、`Quarterly_2026-Q2`。

**模板**：内置日 / 周 / 季 / 年四套模板，也可在设置里导入自己的 Markdown 模板（支持 `{{date}} {{range}} {{entries}} {{summary}} {{stats}}` 等占位符）。

### 4. 配置 AI（可选）

不配置 AI，记录 / 回顾 / 报告「直接整理」/ 导出都照常可用。配置后才有「AI 总结」和「多日拆分」。

进入右上角 ⚙ 设置 →「AI 服务」，填三项：

| 项 | 填什么 |
|---|---|
| Base URL | OpenAI 兼容接口地址 |
| API Key | 云端服务的密钥；本地模型一般留空 |
| 模型名 | 例如 `qwen3:14b` |

- **想完全离线用 AI**：装 [Ollama](https://ollama.com/)，`ollama pull` 一个模型，Base URL 填 `http://localhost:11434/v1`。数据不出本机。
- **想用云端**：填对应服务商的 OpenAI 兼容地址和 Key（需联网）。

填好 Base URL（云端再填 Key）后点「测试连接」——**不用先填模型名**：连通后会在按钮右侧显示结果，并把该服务可用的模型拉成一个下拉框、默认选好一个，你直接在「模型名」里挑即可。改了 Base URL / Key 需重新测一次。

### 5. 主题

设置 → 外观，可选 跟随系统 / 浅色 / 深色，即时生效。

### 6. 数据与备份

- 安装版数据在 `%APPDATA%\com.daylog.app\`；绿色便携版在 exe 旁的 `data\`。
- 设置里可「导出全部 JSON」备份、「导入 JSON」恢复（按记录 ID 去重，不会重复）。
- 卸载不会删除你的数据。

### 快捷键

| 快捷键 | 作用 |
|---|---|
| `Ctrl+1 / 2 / 3` | 直接切到 记录 / 回顾 / 报告 |
| `Ctrl+Tab` / `Ctrl+Shift+Tab` | 在三个页面间循环切换（下一个 / 上一个） |
| `Ctrl+F` | 跳到回顾页并聚焦搜索框 |
| `Ctrl+Enter` | 保存当前记录（记录页输入框、回顾页补记框） |
| `←` / `→` | （回顾页）上一月 / 下一月 |
| `PageUp` / `PageDown` | （回顾页）上一年 / 下一年 |
| `Esc` | 关闭设置 / 弹窗 / 当天详情面板 |

## 开发与打包

- 技术栈：Tauri 2 + React 18 + SQLite。
- 本地预览 UI（无需 Rust）：`npm install && npm run dev`。
- 完整应用 / 打包（需在 Windows 上）：`npm run tauri dev` / `npm run tauri build`。
- 打包产出绿色单 exe + NSIS 离线安装包，完整步骤见 **[PACKAGING.md](PACKAGING.md)**。
- 需求文档见 **[REQUIREMENTS.md](REQUIREMENTS.md)**。
